# Contributing to AIDEM

AIDEM should improve through practical use, critique, and pilot evidence.

## Useful contributions

- Field-tested examples
- Template improvements
- Risk scoring improvements
- Dashboard improvements
- Pilot lessons
- AI-native workflow examples
- Notes from Scrum, Kanban, or hybrid delivery teams

## Contribution principles

1. AIDEM is a grooming and calibration protocol, not a precision formula.
2. Estimates should use ranges, not false precision.
3. AI should not be treated as a blanket productivity discount.
4. Review, testing, rework, flow, and risk should remain visible.
5. The framework should stay light enough for real grooming sessions.

## Suggested issue format

```md
## Problem
What part of AIDEM is unclear, too heavy, or incomplete?

## Context
What type of team or workflow did this appear in?

## Suggested change
What should change?

## Evidence
Pilot data, examples, or observations if available.
```

## Pull requests

Prefer small, reviewable changes. For larger changes, open an issue first.

## Example safety

Do not submit private company details, customer data, production identifiers, or sensitive delivery metrics. Generalize examples before contributing.
